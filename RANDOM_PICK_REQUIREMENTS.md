# 网格视图随机点名 — 需求文档

> 日期：2026-06-12 | 状态：待确认

---

## 1. 功能概述

在座位网格视图中提供「随机点名」功能。教师点击开始后，系统在**已签到学生**中快速随机切换，网格对应座位同步闪动，直到教师点击停止确定人选。

---

## 2. 触发入口

| 位置 | 说明 |
|------|------|
| 网格视图页面顶部（view-toggle 旁边） | 新增 `🎲 随机点名` 按钮 |
| 按钮分为两个状态 | 未启动时显示「🎲 随机点名」，启动后变为「⏹ 停止」 |

---

## 3. 核心交互流程

```
教师切换到网格视图
    → 点击「🎲 随机点名」
    → 网格中已签到座位开始快速轮换高亮
    → 同时弹出一个浮层卡片显示当前选中的学生信息（头像占位 + 学号 + 姓名 + 座位）
    → 轮换速度分段：快（50ms）→ 中（150ms）→ 慢（300ms）→ 停
    → 教师随时点击「⏹ 停止」或按空格键 → 立即停在当前学生
    → 最终选中学生座位持续高亮（金色边框 + 脉冲动画）
    → 浮层卡片同时定格显示结果
    → 再次点击「🎲 重新点名」或关闭浮层 → 恢复普通网格状态
```

---

## 4. 动画效果

### 4.1 网格座位闪动

| 阶段 | 效果 |
|------|------|
| 轮换中 | 当前选中座位显示**金色背景 + 缩放脉冲**，上一个座位立即恢复原色 |
| 已停止 | 最终座位保持**金色边框 + 呼吸动画**（1.5s 周期），区别于普通签到绿色 |

### 4.2 浮层卡片

| 状态 | 行为 |
|------|------|
| 轮换中 | 卡片内容随轮换实时更新，卡片本身带轻微弹性动画 |
| 停止后 | 卡片放大定格，显示完整结果，底部出现「✅ 就是TA」「🔄 重新点名」「✖ 关闭」三个按钮 |

### 4.3 音效（P2，可选）

轮换时发出轻微滴答声，停止时一声确定音。如果实现复杂则不做。

---

## 5. 点名范围

| 规则 | 说明 |
|------|------|
| 点名池 | **仅限已签到学生**（空位不参与、冲突座位不参与） |
| 可重复点名 | 同一学生可被多次点名（不行除已点名过的） |
| 无签到学生时 | 按钮置灰，提示"暂无已签到学生" |

---

```
┌─────────────────────────┐

│      │ 👤   │  头像占位  │
```

轮换中仅显示学生信息不断刷新，"就是TA"按钮高亮。停止后定格。

---

## 7. 与现有功能的兼容

| 现有功能 | 影响 |
|----------|------|
| 网格视图实时更新（SSE） | 新签到进来的学生应即时加入点名池 |
| 列表视图 | 切换到列表视图时自动停止点名 |
| 重置签到 | 被重置的学生立即移出点名池；如果正在轮换中恰好选中该学生，跳过 |
| 二维码弹窗 | 点名中和二维码弹窗互不冲突，可各自打开关闭 |

---

## 8. 技术要点

| 要点 | 方案 |
|------|------|
| 轮换定时器 | `setInterval` 控制切换速度，支持动态调速 |
| 速度递减 | 快（50ms）1.5s → 中（150ms）1.5s → 慢（300ms）直到手动停止或超时（总计约 6s） |
| 空格键停止 | `keydown` 事件监听 `Space`，仅在点名进行中响应 |
| 随机算法 | Fisher-Yates 预先打乱已签到学生列表，顺序遍历（保证每个学生被选中概率均等） |
| 状态管理 | 前端状态机：`idle → running → stopping → result` |

---

## 9. 状态流转

```
 idle ──点击「开始」──→ running ──点击「停止」/空格──→ result
  ↑                        │                              │
  │                        │ 超时（约 6s）自动停止         │
  │                        ▼                              │
  └──── 点击「重新」/「关闭」←─────────────────────────────┘
```

## 10. 待确认事项

1. **点名池范围**：只从已签到学生中抽取，还是要包含名单中所有学生（含未签到者）？
2. **停止方式**：是否支持空格键停止？还是只点按钮？
3. **自动停止**：轮换约 6 秒后是否自动停止，还是一直循环直到手动停止？
4. **速度变化**：三段变速（快→中→慢）可以吗，还是一直同一个速度？
5. **音效**：要不要加滴答音效？
6. **结果记录**：是否需要在某处记录"点名结果"（如某学生被点到几次），还是只做一次性展示？
